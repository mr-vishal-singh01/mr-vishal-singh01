## 🔍 Reporting a Vulnerability

If you discover a security vulnerability within any of my repositories, I appreciate your help in disclosing it to me in a **responsible manner**.

### 📨 How to Report
Please do **not** open a public GitHub issue for vulnerabilities. Instead, send a detailed report to:
**[mrvishalsinghchauhan01@gmail.com](mailto:mrvishalsinghchauhan01@gmail.com)**

**Please include the following in your report:**
- A detailed description of the vulnerability.
- Steps to reproduce the issue (PoC scripts are appreciated).
- Potential impact and suggested mitigation steps.
- Your GitHub handle or name if you wish to be credited.

### ⏱️ Response Timeline
- **Initial Acknowledgment:** Within 48 hours.
- **Triage & Status Update:** Within 7 days.
- **Public Disclosure:** Following the 90-day responsible disclosure window (or earlier if patched).

## 🤝 Responsible Disclosure Guidelines

To encourage security research, I ask that you follow these guidelines:
1. **No Data Leakage:** Do not attempt to access or modify data that does not belong to you.
2. **No Service Disruption:** Avoid performing tests that could cause a Denial of Service (DoS).
3. **Confidentiality:** Keep the vulnerability confidential until it has been resolved and a public disclosure is agreed upon.
4. **Lawful Research:** Only conduct research on systems and networks you are authorized to test.
